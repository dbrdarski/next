# Part A — Constitution

**The nine core principles [decided]:**

1. **Contracts replace surface type annotations** — types survive as the coarsest contracts (Kinds).
2. **Compile-time only.**
3. **Implicit by default, explicit optional** — ratified constitutional [user]: annotations are never mandatory anywhere; inference alone carries the language.
4. **Compositional algebraic structure** — interned compositional expressions.
5. **Code as data** — interned canonical forms; equivalence-under-rules is pointer comparison.
6. **Demand-driven tracking** — demands propagate as themselves and resolve late (C§13).
7. **The compiler itself must provably terminate, and no verdict depends on effort spent** — secured globally by C§13.3.
8. **Three-voice honesty** — proven / refuted-with-witness / unproven.
9. **One deliberate softening, precisely scoped** — unproven **grounding** in warned gray regions compiles with a warning; warn-and-compile exists for no other error class. **Scope clarifications [user, 2026-07-17]:** acknowledgment/suppression covers grounding only, **never safety** — a constraint the future `@suppress` statute inherits; safety-suppression would be a constitutional amendment re-admitting runtime failure language-wide (door shut; boundaries are conform's job). **Trap clause (physics, not semantics):** machine limits (memory, stack) abort execution — the machine's limit, never a Principle 9 entry.

**Surface-governing principles [user, ratified 2026-07-13]:**

- **Semantics over form.** The kernel AST and its semantics are the normative language; surface syntax is temporary form. One official syntax, normative-but-one; alternate syntaxes are far-future views, never peers. Decisions sort into **AST-level** (ruled definitively) and **projection-level** (revisable renderings). The kernel AST specification is the normative contract **[shipped: `next-kernel-ast-specification-v0-1.md`]**. Files-as-AST: far-future shelf.
- **Zero reserved words.** Achieved by mechanism, not fiat: `true`/`false`/`null` are predeclared prelude bindings; surface words (`import`, `export`, `from`, `when`, `where`, `module`) are contextual and position-disambiguated (enabled by no-juxtaposition application and the bare-expression-statement rules); `@`-names are sigil-shielded; `match`/`return`/`if`/`case` do not exist as words. Standing rule: any future word-shaped proposal must state its de-reservation mechanism or be rejected.

**Design heuristics [recorded]:** canonical form over smarter analysis; shelf shapes grown by demand; external hostile review as part of the discipline; equivalences vs proof rules never conflated; per-error-class verdict policies; stated dependencies; facts carry their domains; demands resolve late; soundness rules moonlight as termination rules; mutually dependent facts settled jointly; templates are symbolic, instances are analyzed (whatever depends on a capture — regions, grounding, facts — belongs to the instance, not the shape; constant-parameter extraction is the same idea at every level: expressions, summaries, mutual-recursion groups); uncertainty selects, never deselects (rows not proven disjoint are selected; dependency edges not proven empty exist — the third voice widens the analysis, never narrows it); derivation is law-composition over a demand-grown rulebook — never watching runs, never searching invariants.

# Part B — Language Semantics

## B1. Values and equality **[user]**

All values immutable and eagerly interned — **same value = same pointer, universally [user, restored 2026-07-17]**. Closures intern **shallowly, like data**: key = (canonical-code pointer, capture pointers) — the cost of interning a small tuple; **calls are never memoized** (invocation always runs; only construction dedups); code canonicalizes once, at compile time — no runtime code analysis. μ-group members intern at window close keyed by the group's bisimulation fingerprint; in the **closed** stored form μ-markers are **positional and group-relative** — never back-pointers, and never binding identities (binding-referenced markers are construction-window staging; positional rewrite at close is what lets isomorphic groups intern to one pointer, per the ruled shape identity) — with the group as **one allocation**, members referencing by internal offset; location markers alone stay nominal binding identities, by design. The interned immutable value graph introduces **no owning cycles** — μ-group internal references are non-owning positional offsets (F3) — so value refcounting is collector-free; the separate location/closure ownership graph may cycle and awaits F7's lifetime strategy [claim scoped 1.0.4]. y/z, `Equals(<function>)`, and location-distinctness all hold; `==` is value equality everywhere; there is **no reference-identity operator and no observable reference identity** in the language. Precisely [1.0.2, restoring C§8's asymmetry caveat]: locations are not values and can never be named or compared; closure *identity* nevertheless includes captured locations (C§12.3 layer 1) because they determine future behavior — same-body closures over distinct locations are distinct values. Value types: Boolean, Null, Number, String (UTF-16 storage), Tuple, Record, Function, Indeterminate(form). Aliasing of structures does not exist — only Boxes share fate. Semantic breaks vs JS (same spelling, new truth): structural `==` flips false→true; float artifacts disappear; `1/0` is Indeterminate, not Infinity.

## B2. Numbers **[user]**

Exact. PoC: canonical arbitrary-precision rationals; decimal literals denote rationals (`0.5` ≡ 1/2; `0.1 + 0.2 == 0.3` is **true**). Division is total via Indeterminate. Rational printing: exact fraction form; decimal rendering when the reduced denominator's only prime factors are 2 and 5 — i.e., it divides some power of ten (`1/2` → 0.5, `3/20` → 0.15, `1/3` → 1/3; `0.5` round-trips) [corrected 1.0.2: the inherited “is a power of ten” condition was mathematically wrong]. Irrational-producing operations omitted from the PoC; full-language representation (Phase 1 Reals) **[open]** — the interner sanity bound for astronomical exponent literals is assigned there.

## B3. Stratification of self-reference **[decided]**

Data is structurally acyclic through data fields (`x = [x]` rejected by the binding rule). Recursive reference may pass through function bodies and environments (`x = [() => x]` legal), represented as code self-reference via μ-markers — never cyclic data; the value-graph analysis confirms it — mixed rational graphs cycle only through code nodes (μ package, round 4). Contracts carry self-reference only as named recursive contracts (C§9). Direct cyclic data is illegal; a function stored in a structure may recursively refer to the surrounding binding through code self-reference.

## B4. Bindings, scope, closures **[user]**

Bare `=` is immutable binding; no reassignment; no let/const ceremony. `:=` exists only inside Mutators — declaration and mutation are deliberately two visibly different operations. **Late-binding resolution** is the closure semantics: free variables (including self- and mutual references) resolve at evaluation, not construction; static rule — evaluation may reference only names bound by earlier statements; **lambda bodies are exempt**. Recursion and mutual recursion need no grouping construct (letrec **[dissolved]**). Eager capture is a pure runtime optimization. Free references to box bindings are dynamic reads of the location's content.

**Unbound access [policy settled; mechanism split].** Unbound evaluation is a compile-time error, never a runtime event. Closed [user]: forward *calls* are rejected by the analyzer's environment check at the evaluating statement; diagnostics carry demand origin, reference site, and resolution failure via the provenance side-table (occurrence→span map; interned values carry no spans). **Open-value identity: RULED [user, 2026-07-17] — shape, via strict openness.** A mutual group is its infinite unfolding written in steps: `a = [() => b]; b = [() => a]` *is* `y = [() => y]` constructed in two statements — construction is not identity; shape is. Mechanism: group windows close at the last member's binding; members canonicalize **jointly** into μ-group form; late twins fold in (`a2 = [() => b]` interns onto `a`); **bisimulation collapse embraced** — a shape-symmetric group collapses into the self-loop (`a == b == y`), and truly symmetric slots are one slot (the member-ordering rule's foundation). **The split rule [user, via the lift lens]:** this governs μ-**immutable** markers only; **location markers remain nominal** (fork 13, mandatory — same-body closures over distinct locations are distinct values); mixed closures canonicalize μ-structure by shape around nominal location markers. Self-reference unchanged (window = the executing statement). Frozen per the equality-freeze doctrine. **Open-value observation prohibition [adopted, μ review round 1 — forced by immutable identity]:** openness is contagious (a structure containing an open value is open); open graphs are construction state, never observable values — within the window construction composes freely, but `==`, export, return past the window, call-argument use, or observation by a non-constructing statement is a compile error (extending this section's environment check); all markers resolve simultaneously at window close, containers intern at close, and only closed graphs are values. Equality on values is thereby constant — the nominal-while-open interim rule is withdrawn. The normative mechanism is the μ package (C§11). Theorem for the semantics spec: *in an accepted program, every evaluated reference is bound.*

**Function identity — three layers** (canonical statement C§12.3): value identity (canonical body + resolved immutable captures + μ/recursion markers + Box-location markers); function shape (template key); analysis instance (shape + captured-environment contract tuple).

## B5. The three act-kinds **[user; E5 amendments applied 2026-07-17]**

- **Pure computation** may read box bindings; purity is **snapshot-relative** — referential transparency holds within any Mutator-delimited segment.
- **Mutators** — the only writers of state; synchronous, statement-level, run-to-completion. **Transactional = publication-only [user, 2026-07-17]:** a Mutator's writes publish together at completion; a body that never completes publishes nothing. Rollback machinery is **deleted** — no trigger exists, by the theorem: **Mutators cannot fail** (no IO by definition; unproven safety is un-suppressible; arithmetic no-answers are values — nothing failable can compile into one). Every non-finish (machine trap, gray divergence) is non-completion ⇒ nothing published. **Staging model [1.0.2 — the mechanism publication-only requires]:** a Mutator maintains a pending write-set; box reads inside the Mutator see pending writes (**read-your-writes**, generalizing G1's recorded answer from computeds to all reads); completion atomically publishes the set; within an Effect, a completed Mutator statement's published state is visible to all subsequent statements — ordinary sequencing, dependency (5). **Nested Mutator calls join the caller's transaction [user, 2026-07-17]** — the mutator boundary *is* the transaction; one publish per outermost call; a separate publish would break the caller's atomic boundary. Shelf note: Oddo's `transact` name stays available if an effect-world grouping form is ever wanted (no field precedent either way — Oddo has no nested-mutator tests); an explicit `@transact` resident was considered and rejected — it would make the unwrapped composition the silently broken default and mint a second concept where `@mutate` already is the transaction marker. **World admission matrix [1.0.3 — the explicit form the cannot-fail theorem requires]:** pure world may call pure only; mutation world may call pure and Mutators (joining, per the ruling) and **never Effects**; effect world may call all three; for a union of possible callees, every possible act-kind must be admitted by the current world — with the witness discipline [aligned 1.0.7]: a represented, **inhabited** alternative with a disallowed act-kind **refutes**; a coarsened/unknown possibility without a realizable witness is **unproven** (still blocking at a safety seat, never a manufactured refutation). **World context governs immediate expression evaluation; a nested function's body is analyzed in the world its own declared act-kind names [1.0.7]** — a pure lambda created inside an Effect is a pure function; lexical position confers no permissions. **Gray-in-Mutator is Principle 9's sharp corner:** a diverging suppressed-gray Mutator hangs the single mutation branch — covered by the existing gray warning; no new semantics. Compound mutation operators (`+:=`, `-:=`, `*:=`, `/:=`, `%:=`, `**:=`, `&&:=`, `||:=`, `??:=`) are sugar over `:=`; **path mutation** `user.name := v` is sugar for read → immutable functional update → box write, atomic within the Mutator. **Returns: current law is return-nothing; [LEANING — user, deliberately unstamped, revisable]: Mutators return** (design B's other half); nothing downstream builds on it either way.
- **Effects** — the only writers of the external world; may read state and perform IO; never write boxes except by invoking Mutators; excluded from expressions in pure world. Causality hygiene: *effects observe state, Mutators change state, effects never cause what they observe except through Mutators.* **Callable form [user, 2026-07-17]:** `@effect name = (...args) => { }`, body law fully inherited (see E14). **Failure story: B6.**

**Dependencies [decided]:** (1) Mutators and Effects cannot appear in expressions, guards, or contract predicates *in pure world* — see the containment asymmetry; (2) **pure** functions cannot transitively invoke Mutators or Effects [worded 1.0.8 — Effects themselves produce `Union(T, Failure)` values]; (3) canonicalization never moves box reads across a Mutator boundary; (4) a Mutator's writes publish atomically at completion or not at all; (5) box state is stable during any ordinary synchronous expression.

**Containment asymmetry [user-corrected]:** act bodies contain (pure) expressions — logical operators, ternaries, `~`-seats, guards all available inside Mutators/Effects; the reverse direction is **world-law, stated precisely [1.0.7]**: act *declarations* are statements, never expressions; act *calls* are ordinary application nodes whose legality the current world's admission matrix governs — pure world admits only pure callees, so pure-world expressions are act-free by jurisdiction, exactly as the inline-call ruling requires. **Act-call placement [user, 2026-07-17]:** inside act bodies, inline act calls are legal surface and mean exactly what evaluation order says — occurrences fire when evaluation reaches them, left to right; written twice = fires twice; untaken branches don't evaluate. **No transformation exists** (the "act-body hoisting desugar" was withdrawn as never-designed); pure world remains act-free by *jurisdiction* (the world model, E13), not by a body-crawling checker.

## B6. External failure **[user, RESOLVED 2026-07-17 — fork 14 closed]**

**Model (i): representable external failures are ordinary values.** An Effect call's result binds in act context as `Union(T, Failure)`; direct access on the unnarrowed union (`data.body`) is a **compile error**; a match narrows first — contracts-as-patterns is the discharge surface. ## B7. State surface **[user]**

`@state` and `@mutable` are both real and distinct: both declare Box locations with identical contract treatment (C§12.4); the difference is **reactive scheduling** — the 2×2: `@mutable`/`@state` are the non-reactive/reactive **state** declarations; `@effect`/`@reactive` are the non-reactive/reactive **effect** forms. Writes to `@state` trigger revalidation of `@computed`/`@reactive`; writes to `@mutable` trigger nothing; `@effect` is the plain Effect act-kind, never scheduled by the reactive layer; `@computed` is a pure derivation over state reads (an ordinary pure function whose re-evaluation timing is runtime scheduling; no non-reactive twin needed — an ordinary pure binding is that). **Default mutator [user, 2026-07-17]:** a `@state` declaration can come with its auto-derived assignment mutator (`@mutate setX = (v) => { x := v }` conceptually) — a full mutation-world citizen, equality-guarded (exact under interning: writing an equal value provably fires nothing); the Oddo tuple spelling `[x, setX]` dies [user: ugly]; replacement spelling queued; statute details (shape, always-vs-opt-in, references-as-values) parked with the `@` block. The reactive *layer* (dependency tracking, scheduling, UI integration) is **[fenced]** — Part G1 is its opening file.

Atomicity/stability rules hold by construction, not locking. **[fenced — fork 15]:** the entire cross-thread surface (functions-in-messages, actors, message-passing rulings); dossier: option (y) — binding-free functions may travel, an ordinary demand at send seats using binding-identity — is the recorded stronger candidate over (x) blanket ban; trap scope in multi-worker programs (kill the worker or the program; supervision) joins the same fence.

## E4. Hasks — expression-to-function conversion **[user]**

**(1) Modules are a distinct namespace construct — not value-layer Records [user amendment, 2026-07-17, round two; supersedes the modules-are-records clause].** The reason is structural: ordinary Records hold values and cannot hold Boxes; module entries are **bindings** — immutable or Box (`@state`/`@mutable` exports). `M.name` resolves the exported binding — for a Box binding, an ordinary dynamic read of current content (B4/C§12.3); the access surface is record-like; the location is never a value. **Enabled, not restricted [user correction, in-session]:** the amendment exists so modules *can* carry state. **Imports deliver bindings** — `import { count } from Counter` binds the state binding itself; references are live dynamic reads; imported state is exactly the cross-module reactive dependency (per-export knowable under static modules — G1 finding 8). **Module references alias** — `m = Counter` is namespace aliasing under whole-program resolution; `m.count` resolves through the alias to the binding, live, identical to `Counter.count`. **Spread-composition stands as ruled** — `export Geo2 = { ...Geometry, describe }` evaluates entries as references at spread time (Box bindings → content, C§12.4), yielding an ordinary Record. **[open — narrow corner only]:** a module in a true value seat (stored as a structure field; module equality) — undefined for now; nothing depends on it. `export` prefixes a binding, one way only; no default exports, no export lists. Circularity resolves as mutual recursion does (late-bound lambda bodies); circular value-level dependence unconstructible. **(2) Headers on export.** `module Name` (dotted names legal) as the file's first statement — required iff the file exports anything; entry points that export nothing carry no header and are simply unimportable. # Part F — Runtime Architecture

## F1. Core model **[prior-conv, user]**
Acyclic closed values — closures included, shallow-keyed per B1 — intern **eagerly, bottom-up** into hash-cons tables; open recursive groups intern **jointly at window close** through the mixed-graph quotient (C§11) [wording scoped 1.0.4]. Bottom-up mechanics: construct → hash (shallow — children are already interned pointers) → probe → return existing pointer or insert. Same value = same pointer, always; for existing interned values, equality and cached-hash access are O(1) [claim narrowed 1.0.2]; canonical construction costs the immediate representation (code units for Strings, arity for Tuples/Records, integer magnitude for Numbers); structural sharing and memory deduplication automatic. **No tracing collector for the immutable interned value graph** — external-owner reference counting suffices there, the graph acyclic by construction [scoped 1.0.7; the prior whole-runtime “No GC” headline overstated]; the location/closure ownership graph is a separate question: a closure may capture a Box location, and a location's content may be a closure capturing that same location (`callback := () => callback`), so escape lifetime and location-cycle strategy need their own model [1.0.3 — filed to F7 for the runtime hostile round]; the interning table's membership is **not** part of the reference count (external-owner counting) [1.0.2 — the inherited “strong references” wording made zero unreachable]; the zero transition removes the entry and frees, atomically, mediated by the interning writer thread (F6) — also what stands between lock-free readers and removal; the precise reclamation scheme is an F7 open.

## F2. Memory layout and per-type tables **[prior-conv]**
Uniform value layout: `[ ref_count | hash | tag | header | fields… ]` — hash cached inline; fields are primitives or interned pointers. Per-type tables: **String** (UTF-16 code units); **Tuple** (pointer sequence, no header); **Record** (shape pointer + value sequence; a shape is itself an interned Tuple of sorted key Strings — no separate shape table); **Number** (PoC: canonical reduced BigRational `[rc | hash | tag | num | den]`, arbitrary-precision components). **Function table [user, restored]:** closures intern shallowly — key = canonical-code pointer + capture sequence (all pointers; small-tuple cost); the code side interned at **compile time** (canonical forms, C§11), which is what makes code-identity a pointer test; calls never memoized. Whether the per-type tables unify into one tagged table: **[open]**, semantics unaffected.

## F3. Function values under the three-layer identity
The runtime implements **value identity** (C§12.3 layer 1): body/code pointer + resolved values for immutable free names + **construction-window binding references, rewritten at window close to positional group-relative μ-markers** [worded 1.0.7 — closed values never carry binding identities; shape-isomorphic groups intern equal only because those are erased] + **location markers** for box-binding references (nominal, alone, by design). Markers, not back-pointers, keep the **interned immutable value graph** pointer-acyclic [1.0.5 — scoped; the location/closure ownership graph is F7's separate question] — a function value never owns a pointer into the value being constructed around it, so ref counting needs no cycle collector even for `x = [() => x]`. Same-shaped self-referential structures **intern equal** (rational-tree μ-comparison over canonical code, fingerprinted at window close); function equality is thereby a pointer test like everything else [user, restored]. Shape and analysis-instance identities (layers 2–3) are analyzer-side tables, not runtime objects.

## F4. Boxes are locations **[user — supersedes the prior conversation's open Box design]**
No Box value and no Box interning table. A box is a mutable **slot** with runtime-internal identity; its content is always an interned value; referencing the binding reads the content; contents-comparison is ordinary value `==` of two reads. Reactive scheduling hooks attach to slots in the future reactive layer. Locations are introduced by `@state`/`@mutable` declarations.

## F5. Reference counting **[prior-conv]**
**Two-level ref counting:** thread-local counts (free) + shared counts, coordination only at zero-boundary crossings. **Deferred decrement** via a pending-free queue (avoids deep cascade stack overflow on large teardown). Table removal atomic with free.

## F6. Concurrency **[one philosophy used twice]**
**Interning:** dedicated single-writer thread for table insertions; lock-free reads; construction misses round-trip to the writer, hits are local. **Mutation:** the single mutation branch (B8) — the same architectural move applied to state. **Messages:** sending an interned value across threads is pointer transfer — effectively zero-copy, structural sharing included; the two-level ref counts are exactly the machinery cross-thread sharing needs. Values only; boxes cannot be sent (automatic); function-in-message is the fenced ruling (B8).

## F7. Deferred runtime items **[open]**
Hash function choice; table structure (open addressing vs chaining); per-type table unification; compile-time interning of static values + static shape inference (compiler optimization pass); precise deferred-decrement policy; wasm-target memory strategy; the lock-free-read reclamation scheme — writer-mediated grace, epochs, or hazard pointers [1.0.2]; the μ-group **member handle** — interior pointer vs canonical (group, slot) pair; the literal same-pointer claim must match the chosen form [1.0.4]; the Box-location lifetime model — escaping captured locations; location→closure→same-location ownership cycles; candidates: mutation-branch-lifetime slots, region/arena, owning handles + a cycle strategy [1.0.3]. The function-children congruence corner is **retired** — closures intern, children have canonical pointers [user ruling, restored interning].
