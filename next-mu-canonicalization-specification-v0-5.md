## 1. Two graph domains (the split the three-layer model demands)

**Code graph** (Algorithm A's domain; produces **shape**, layer 2): atoms are `ParameterRef(i)`, `CaptureRef(i)` (a positional placeholder — never a value), `μRef(d, s)` (serialized form only — two-phase rule, §5), `Primop(tag)`, `ActKind(kind)`, `Literal(payload)` (literals *in the body* are code), `FieldName(name)`. No data pointer and no location identity ever enters canonical code — `makeAdder(1)` and `makeAdder(2)` share one shape by construction.

**Value graph** (Algorithm B's domain; realizes **value identity**, layer 1) — **mixed node grammar [round 4]**: `ValueNode = ClosedDataAtom(pointer) | LocationAtom(bindingIdentity) | ClosureNode(bodySkeleton, captureEdges[]) | TupleNode(elementEdges[]) | RecordNode(canonicalShape, fieldEdges[])`. Closed, already-interned children are atoms (fast leaves); locations nominal (fork 13), never unfolded, never merged. Tuple/Record nodes exist because **open aggregates participate in groups**: the flagship `x = [() => x]` is `TupleNode([ClosureNode(body, [edge → TupleNode])])` — a finite rational graph whose cycle passes through code, never through data fields alone; B3's stratification stands verbatim.

## 2. Grouping and windows

μ-grouping is by **strongly connected component** of the scope's reference graph (lambda-body references included). Members need not be adjacent: the **window** spans from the first member's statement to the last member's; unrelated statements may interleave, subject to §4. A lone self-reference is an SCC of one, closing at its own statement (the standing rule).

## 3. Normal-form laws

Laws 1–3 and 5 as v0.1 (vacuous erasure; adjacent-binder merge; minimal group = the reference-SCC; canonical slot order per §5's ranking). **Law 4 — slot dedup with capture routing [revised, round 2]:** the canonical group template carries a **group-level capture space** and per-slot **routing maps** — `GroupTemplate { groupCaptures[]; slots: [{ body, captureMap: localRef → groupPosition, internalRefTargets }] }`. Slot bisimulation compares body structure, recursive-target structure, **and the routing map**; slots merge only when symbolically identical *including routing*. The makePair counterexample — `a = n => n == 0 ? x : b(n−1)`, `b = n => n == 0 ? y : a(n−1)`, symmetric bodies routing their base capture to different group positions — therefore does **not** collapse at code level (merging would destroy the alternating capture behavior). **The two-level distinction:** *code-level* dedup requires identical symbolic routing; *value-level* dedup may additionally exploit equality of actual captured pointers — `makePair(v, v)` instantiates members whose value graphs are bisimilar, and the value interner legitimately collapses them (same infinite tree) while the code template stays two-slotted. The ruled collapse (`a == b == y` for the capture-free group) is unaffected: empty routing maps are identical by vacuity. **Capture-space order [round 3]:** `groupCaptures[]` is ordered by each captured binding's positional (de-Bruijn) identity in the containing scope — source-discovery order and member order never participate, so member permutation cannot renumber captures or perturb routing maps (MU-06's invariance depends on this). Value-level collapse operates on the **instantiated group-value graph** (§6). Law 6 (value trees) now reads over §1's value graph. Law 7 (contracts): named recursive contracts canonicalize under laws 1–5 **jointly with the ordinary constructor normal forms** (C§11's sorted/theory-aware canonicalization) — commutative-spelling variants must collapse before or with μ-minimization, never after. **Law 8 — capture vector:** the shape's capture arity and order are the *surviving* `CaptureRef`s of the normalized body, in positional order; captures unused after normalization are dropped from shape and value identity alike (an unobservable capture cannot distinguish values).

## 4. Open values — construction state, never observable **[adopted round 1, with containment scoping]**

**Openness is contagious:** a graph containing an unresolved reference is open, and any structure containing an open value is itself open. Within a group's window, construction composes freely — `a = [() => b]` legally builds an open tuple over an open closure; that is the mechanism, not a violation. **Prohibited while open** (a compile error, extending B4's environment check): `==` or any identity observation; export; return past the window; use as a call argument; observation by any statement that is not itself group construction. **At window close, all markers of the component resolve simultaneously**; containers intern only then (their interning is deferred to close); only the resulting closed graphs are values. Consequence: equality is total on values and *constant* — the v0.1 "nominal while open" rule is withdrawn as the defect it was (it made `==` on immutables time-varying). Compendium B4 carries the amendment.

## 5. Algorithm A — eager code canonicalization (per SCC, compile time)

1. Resolve names positionally; represent group references as **`GroupRefEdge(depth, target-slot)`** — graph edges, never index-labeled atoms (labels would prevent bisimilar slots from merging); collect each slot's external captures into the group capture space, recording the routing maps (law 4). **Two-phase terminology:** precanonical graphs contain `GroupRefEdge`s; serialized `μRef(d, i)` exists only after step 4's ordering — an initial slot number is never semantically meaningful.
2. Apply laws 1–3 and the ordinary constructor NF (law 7's joint pass).
3. **Partition refinement** to bisimulation-minimality over the labeled ordered graph: refine on `(tag, arity, [(edge-position, target-class)…])` — positions are edge labels; `subtract(a,b)` never merges with `subtract(b,a)`. Refinement cost O(E log N).
4. **Canonical slot order** — the ranking algorithm: totally order the minimal quotient's slot classes by memoized lexicographic comparison of their regular trees — compare `(label, arity)` then children classes recursively, memoizing class pairs; two *distinct* minimal classes are non-bisimilar, hence distinguished within ≤ (class-pair count) steps. Uniqueness holds by the v0.1 theorem (a tie ⇒ automorphism ⇒ bisimilar ⇒ already merged). Cost polynomial in the quotient; **no single headline bound is claimed for A overall**.
5. Serialize with `⟨d, i⟩` references; intern. Canonical code pointers are stable **within the language-semantics/canonicalizer namespace** (C§13.4) — not "forever."
