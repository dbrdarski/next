## 1. The application transfer rule

`analyzeOperation(application, AC_operands)` at a seat in world *w* — **the operand state is one joint correlated AnalysisContract [round 5]** denoting `[callee, …arguments]` tuples; the list spelling `[AC_callee, AC_arg…]` remains surface shorthand for its fully-uncorrelated case. **Processing is per live correlated alternative:** extract that alternative's callee metadata → admission (step 1) → input obligation (step 3) → row selection with that alternative's argument tuple (step 4) → join the alternative outcomes (step 7). A projecting implementation is legal but priced: any failure manufactured only by a synthesized cross-pair is **unproven, never refuted**, unless joint reachability is independently proved (AP-29). Steps:

1. **Act-kind admission** — the one **world-dependent** step, applied **at the seat, outside the cache (§7)**. Over `Known(S)`: every member of S admitted by *w* → proven. An inadmissible member refutes **only with an inhabitance-backed witness [round 4]** — an analysis instance is not by itself a runtime closure: a member whose concretization is **proven empty is dropped from S** (metadata normalization, §2); an inadmissible member with an available **represented closure witness** → refuted(witness); an inadmissible member of **uncertain inhabitance** → unproven. Input-obligation refutations carry the same requirement: the witness is a represented (closure, argument-tuple) pair, never an argument alone. Over `Unknown`: **unproven** — no witness can exist; coarsening loses completeness and never invents evidence. Over `Known(∅)` [verdict named, round 3]: no represented application exists — the admission and obligation checks pass **vacuously**; the seat's unreachability is the emptiness machinery's diagnostic, not admission's refutation — a sound vacuous pass, never a manufactured error.
2. **Instance resolution.** `Known({i})` / `Known(S)` proceed per instance; `Unknown` → **unproven at the seat** (the coarsening row); `Known(∅)` as above.
3. **Input obligation — relational over E × A [round 5], per instance and per alternative:** the requirement is `(E × A) ⊑ᴬ AcceptedDomain(instance)` — equivalently **∀ e ∈ γ(E): A ⊑ AcceptedInputs(instance, e)** — universal over environments, exactly as step 4's selection already is; an existential reading (“A intersects some accepted domain”) or a union over environments is non-conforming: it would bless a call some represented closures reject (`x <= limit` under `limit ∈ Range(1,5)` with `Equals(3)` — accepted by some realizations, rejected by others → not proven). **Precisely [round 6]:** the semantic requirement is `R_alt ⊑ᴬ AcceptedDomain(instance)` where `R_alt ⊆ Environment × Arguments` is the relation the live joint alternative induces; **`E × A` is the reference product approximation** (`R_alt ⊆ E × A`) — sound and possibly stronger, so a relational implementation may prove where the product rule lands unproven; preserving joint operand correlation does not require discarding environment–argument correlation inside an alternative — the product rule merely permits it. Spreads via the tuple family; refuted with a jointly represented witness / unproven as before.
4. **Row selection over E × A** — `(E × A) ∩ Row` not proven empty; certificates universal over E; the `Range(1,5)` tri-region flagship (AP-08) unchanged.
5. **Result — `ApplicationOutcome { produced: AnalysisContract; completionWithoutValue: ProvenAbsent | ProvenPresent(witness) | UnprovenPossible; mayNotComplete: Bool }`** [tri-state, round 4 — a Boolean erased the three-voice distinction between a witnessed fall-through and a merely-undisproven live row; `mayNotComplete` stays Boolean because it feeds no safety verdict]: **each selected row contributes its complete kernel outcome summary**, and the flags derive from selected rows **with their evidence [rounds 5+7]**: a fall-through row contributes `ProvenPresent(witness)` **only when `witness ∈ R_alt ∩ Row` is proved** — joint-relation inhabitance, never mere product inhabitance [round 7: a pair in `(E × A) ∖ R_alt` is a synthesized cross-pair, and certifying it would let an expecting seat refute with a witness the program never represents — contradicting the completion-refutation theorem and AP-29's own rule]; a row whose `(E × A) ∩ Row` is proven inhabited **without proved joint membership** contributes `UnprovenPossible`, as does a row retained only because its intersection is not proven empty; `ProvenAbsent` holds exactly when `(E × A) ∩ Row = ∅` for every fall-through row — the product legitimately proves **absence** (since `R_alt ⊆ E × A`), it just cannot prove **presence**. Row selection itself remains product-based; produced-result union and conservative `mayNotComplete = true` remain sound. The same convention governs `mayNotComplete`: `false` certifies absence; `true` is conservative possibility and needs no divergence witness. Act-kind never manufactures a flag. (Under current return-nothing law, every Mutator row's summary *is* `CompletedWithoutValue` — by law, not by kind-inference; the rule stays correct if the returns-leaning ever stamps. Effect bodies carry the same E10 fall-through permission and contribute the flag through their rows like anyone.) **`produced = Bottom` only when the absence of every Produced outcome is proven [round 4 — the gray rule was unsound]:** empty selected domain, all selected rows fall-through-only, or all selected rows proven non-completing. **Grayness never implies Bottom** — gray means grounding is *unproven*, and real executions may produce (`f = n => n == 0 ? 1 : f(h(n))` with `h` ungrounded still reaches 1): the sound gray-result rule is the **coarse fallback** — the best result contract derivable *without* the induction license (a verified `where` return bound where one exists; otherwise the erased/non-inductive derivation; at minimum Top) — with `mayNotComplete = true` riding along; partial-correctness inference for gray remains the standing open, so the reference path never sharpens this fallback by inference. **`mayNotComplete`** means exactly: at least one execution represented by the summary may not complete — aligned to E10's triple, gray recursion being its usual but not defining source.
6. **Completion demand — three-voiced at the seat [round 4].** Expecting seat: `ProvenAbsent` → proven; `ProvenPresent(witness)` → **refuted, with the witness**; `UnprovenPossible` → **unproven** — never a manufactured refutation. Statement seats accept all three. `mayNotComplete` violates nothing anywhere. (An implementation may keep a Boolean inside an over-approximate summary; the seat judgment still requires the tri-state.)
7. **Union of callees.** Obligations conjunctive; summaries join componentwise — produced by union; completionWithoutValue by evidence-preserving join (any ProvenPresent dominates with its witness; else any UnprovenPossible; else ProvenAbsent); `mayNotComplete` by or; metadata joins per §2's lattice. **The empty join's identity element is Known(∅)'s cached core [round 4]:** `produced = Bottom` (legitimately — absence proven vacuously), `completionWithoutValue = ProvenAbsent`, `mayNotComplete = false`, selection empty. **Memoization — the seat/world split [round 3]:** the cache stores the **seat- and world-independent core** only — per-instance obligation verdicts, row selections, the outcome summary, and the callee act-kind set; **admission (world-dependent, step 1) and the completion demand (seat-dependent, step 6) are applied at each seat from the cached components, never cached with them.** One expression reached from two worlds shares its core and diverges only at the seat-applied judgments (AP-20); putting *w* in the key would be sound but wasteful and is rejected.

## 2. AnalysisContract — the formal abstract domain **[rebuilt, round 2]**

**Two-level semantics.** `erase(AC)` = the ordinary contract C — the *language* denotation ⟦C⟧, untouched. **`γ(AC) ⊆ ⟦erase(AC)⟧`** — the *analyzer concretization*: the runtime values the complete AnalysisContract represents, metadata included. For function positions: **γ in full, jointly constrained [round 4]:** `γ(C, Known(S)) = ⟦C⟧ ∩ ⋃_{i ∈ S} γInstance(i)`; `γ(C, Unknown) = ⟦C⟧ ∩ Functions` at function positions — whence `γ(Bottom, M) = ∅` and `γ(C, Known(∅)) = ∅`, and ordinary-contract narrowing and metadata narrowing constrain **together** (also the clean ground for §1's inhabitance rules). **Normalization [round 4]:** `(C, Known(∅)) → Bottom` and `(Bottom, M) → Bottom` — one canonical key per empty concretization. `γInstance(i) = { closures of i's shape whose captured environment ∈ γ(i.environment) }`, with `γ(instance(shape, env)) = { closures of that shape whose captured environment ∈ γ(env) }` — γ recurses through **annotated** environments (captures may themselves carry function metadata) — and its recursion is acyclic **by the μ package's own structure, not by finiteness [round 4 — a finite graph can cycle]**: self and group references live in the shape's μ-structure (GroupTemplate internal edges), so **annotated environment tuples contain only outside-group captures** — the reviewer's Choice A, which the standing design already is. Should any implementation surface residual cyclic metadata, γ's meaning there is the coinductive (greatest-solution) reading, decided by the μ package's bisimulation membership — Choice B as the safety net, never the reference path. `γ(Unknown)` at a function position = all functions of the erased contract. **Soundness obligations are discharged over γ, not over ⟦erase⟧ — and they are the full verdict set, per seat world [round 4], not an output theorem alone:** **(Admission)** proven ⇒ every v ∈ γ(AC_callee) has an act-kind admitted by w; refuted(witness) ⇒ witness ∈ γ(AC_callee) and inadmissible in w. **(Input)** proven ⇒ for every **jointly represented** pair `(callee, args) ∈ γ(AC_operands)`, callee accepts args; refuted(callee, args) ⇒ **(callee, args) ∈ γ(AC_operands)** — the pair jointly represented, never two independent memberships — and that callee rejects those args [round 5: the separate-membership condition could certify a false cross-pair as a witness]. **(Outcome — joint [round 6]):** `ConcreteOutcomes(γ(AC_operands)) ⊆ ΓOutcome(summary)` — for every `[callee, …args] ∈ γ(AC_operands)`, every concrete outcome of applying that callee to those args is represented — over the **entire** E10 space: every `Produced(v)` has v ∈ γ(produced); every concrete `CompletedWithoutValue` is represented by the tri-state (ProvenAbsent excludes them); every non-completion by `mayNotComplete`. The projected `AC_callee` remains legitimate **for act-kind admission alone** (admission depends only on the callee); outcome semantics are joint — the projected form demanded the summary cover AP-29's false cross-calls that the transfer rule itself excludes. **(Completion refutation)** ProvenPresent(witness) ⇒ the witness is a **jointly represented** completing-without-value execution. Unproven verdicts assert nothing — the third voice — C§16's obligation 3 for annotated operands, per world. Metadata is thereby part of the abstract domain, not trusted procedural side information.

**The metadata lattice [round 2]:** `InstanceMetadata = Known(FiniteSet<Instance>) | Unknown`. `Known(∅)` = **no function is possible here** (an impossible branch — feeds emptiness); `Unknown` = a function is possible, origins coarsened away. Operations: `Known(S) ∪ Known(T) = Known(S ∪ T)`; `X ∪ Unknown = Unknown`; `Unknown ∩ Known(S) = Known(S)`; `Equals(<function>)` refines `Unknown` to its known singleton. **Conjunction, split honestly [rounds 5–7]. Semantic intersection is denotational:** `SemIntersect(A, B) = γ(A) ∩ γ(B)` — a set of values, **not automatically an element of the representable domain** [round 7: the AC domain is not claimed intersection-closed]. **An exact representable meet exists precisely when some AnalysisContract M has `γ(M) = SemIntersect(A, B)`.** **The analyzer transfer is `intersectA(A, B)`**, sound by containment only: `SemIntersect(A, B) ⊆ γ(intersectA(A, B))` — returning an exact representable meet **when the fixed intersection rules construct and certify one** [0.8.1: proveSubcontractA is incomplete by design, coverage may land unproven where inclusion semantically holds, and the inventory is bounded by the fixed rule inventory, not by all representable contracts — the analyzer is never a semantic oracle], else a conservative representable over-approximation. Its results, in preference order: **(i) an exact representable meet constructible and certifiable by the fixed rule inventory** — via `meetInstance(i, j)` (empty if shapes differ; else, **when the fixed rules construct and certify an exact representable environment meet `M_env`**, `instance(shape, M_env)`; empty only when the environment intersection is **proven** empty [round 5 — raw key intersection was unsound: `{Equals(1)‑inst} ∩ {Range(1,5)‑inst}` has nonempty γ yet empty key intersection]) with coverage normalization (`s ⊑ᴬ t` ⇒ retain s; `t ⊑ᴬ s` ⇒ retain t; only genuine partial overlap mints an environment meet) — **only this result is called the meet**; **(ii) a conservative same-shape approximation**; **(iii) `Unknown`** — (ii) and (iii) are over-approximations of the concrete intersection, not meets: they satisfy neither lower-bound law, and **no proof may apply lower-bound or idempotence reasoning to a fallback result** — fallback narrowing may *broaden* the abstract state, soundly [round 6]. `Known(∅)` only ever with a disjointness proof. **Boundedness:** intersections arise only from the fixed contract/narrowing rule inventory; the interned-conjunction alternative `ConjAC(A, B)` (canonical pairs over the advance-bounded inventory) remains the recorded representable-precision option — making it **mandatory** would close the domain under intersection at the price of normative canonicalization and inventory treatment; v1 keeps it optional [round 7]. Narrowing uses `intersectA`; construction unions positionally within an alternative.

**Participation** (as v0.2): environment-tuple identity, evaluation-cache identity, argument tuples, ∪/∩/narrowing, positional identity, fact candidates — every analyzer-state key is annotated. **Correlated alternatives keep correlated metadata [round 4]:** metadata attaches *inside* each correlated union alternative; positional metadata unions are legal only where the ordinary-contract representation has already, deliberately, discarded the correlation — the no-flattening precision rule extended to metadata (flattening `{callee: f_num, arg: Number} | {callee: f_str, arg: String}` positionally would mint false cross-pairs and reject valid programs). **The annotated subcontract [round 4; split round 5]:** the **semantic order** is `AC₁ ⊑ᴬ AC₂ ⇔ γ(AC₁) ⊆ γ(AC₂)`; the **analyzer judgment** is `proveSubcontractA(AC₁, AC₂) → proven | refuted(witness) | unproven` — a sound, deliberately incomplete three-valued procedure for that order (its components are incomplete by design), built from: ordinary-contract inclusion × **metadata coverage** × structural recursion through annotated tuple/record positions × recursive AnalysisContracts via C§9's progress-guarded machinery lifted to annotated form × correlated alternatives per the rule above. Coverage: `Known(S) ⊑ Known(T)` when every source instance **not proven empty** is covered — the triage is explicit [round 5]: proven-empty sources are ignored; not-proven-empty sources require target coverage; **uncertain inhabitance is never silently skipped** (it requires coverage, or the judgment is unproven) — by a target instance of the same shape whose annotated environment covers the source's (⊑ᴬ recursively — so `instance(shape, Equals(1)) ⊑ instance(shape, Range(1,5))` despite distinct keys); `Known(S) ⊑ Unknown` proven; `Known(∅) ⊑ X` proven; `Unknown ⊑ Known(T)` generally unproven. Three-voiced throughout; this procedure joins the tuple family in the executability modulo. Survival, the origin theorem, coarsening-as-implementation-liberty (now: coarsening = replacing `Known(S)` by `Unknown`, never by `Known(∅)`), and the ascription-laundering prohibition all carry forward.

## 4. The instance-chain cutoff

**4a. The admitted instance inventory — constructed, traversal-free [round 2].** The inventory is the projection onto instances of a finite state closure over `InventoryState = (analysis instance, active shape sequence)`:

1. Seed with the instances reachable from the program's nonrecursive/root demands, each with its shape appended to an empty sequence.
2. From each state, symbolically enumerate its possible call transitions (over the instantiated table — annotated). **Transition enumeration includes every instance the fixed metadata-intersection and narrowing rules can produce [round 6]** — meet-derived instances are admitted through this ordinary non-cutoff path and belong to the predetermined inventory; the fixed rule inventory supplies the finite bound (an implementation must not build the inventory from pre-meet instances and then mint an unadmitted one during narrowing).
3. Target shape **not** in the active sequence → **admit the target instance**; continue from `(target, sequence ++ target.shape)`.
4. Target shape **already** in the sequence → **no admission through that path**; the cutoff ladder applies at analysis time.

Finite: no admitted path repeats a shape, so path depth ≤ the program's shape count; the state space is large but **computably bounded in advance** — a closure, never an effort-dependent fixpoint. This construction *is* the definition of "an instance some non-cutoff path admits," and order-independence holds because the inventory depends on the program alone. (Equivalent formulation, implementer's choice: one combined closure over instance-path states, fact candidates, and edges.)

**The ladder** (unchanged in substance): **(a)** demanded `(shape, annotated-env-tuple)` ∈ the inventory → intern the fact candidate + dependency edge; proof settled by §6; input domain must cover; shape alone never suffices. **(b)** a generalized fact per 4b. **(c)** unproven. Caches discharge candidates; they never create or authorize them.

**4b. Generalized facts — the finite basis [round 2].** Ladder (b) is admissible only over **`GeneralizationDomains(shape)`** — an advance-bounded inventory of candidate environment domains derived from the fixed extraction-rule collection and the finite program (constant-parameter extraction's affine/enumerable dimensions being the seed rules); only members may become generalized candidates, so the candidate universe stays bounded with (b) enabled. A generalized fact is `(shape, EnvDomain, I, C)` where **EnvDomain is annotated**; **I indexes the region table of the template instantiated over EnvDomain itself** — its rows are subsets of `EnvDomain × Arguments`, never row identifiers borrowed from a concrete instance; grounding and return certificates hold universally over `γ(EnvDomain)`. Omitting (b) remains the honest reference default until an implementation carries this kit.

## 6. The candidate graph

**Closure, reworded [round 2]:** seed with the candidates the program's safety obligations demand; for each newly discovered candidate, **construct its symbolic verification obligation** — evaluate the body symbolically, build F_I, discover its recursive fact references, **intern every referenced candidate and edge** — repeating until no unseen candidate remains; *no verification occurs during discovery* (premature unproven results are non-conforming). Then collapse SCCs and process in reverse topological order (dependencies first); per cyclic component, one joint vector pass. **Failure kinds:** vector failure → all members unproven (per-compilation); **individual semantic refutation** requires a **realized completing witness [rounds 2+4]**: a triple `(e, x, v)` with `e ∈ γ(instance.environment)` (for generalized facts, `e ∈ γ(EnvDomain)`), `x` in row-set I **under e**, the realization (shape, e) applied to x completing with `Produced(v)`, and `v ∉ γ(C)` — execution isn't determined by x alone when captures vary, and a non-completing input is never a witness against a return bound — such a candidate is refuted permanently in-namespace, with the witness. Edges identify their full targets; order-independence per §4a's construction, tested across demand order, cold/warm caches, worklist and traversal orders (AP-10).
